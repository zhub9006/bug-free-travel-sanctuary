# 🤝 Contributing Guide

> How to add a destination or seasonal data update to **Bug-Free Travel Sanctuary**.

## 1. Fork & Branch

```bash
git checkout -b feat/destination-name
```
Convention: `feat/<name>` for new destination, `fix/<topic>` for updates.

## 2. Fill Template

- Copy `guides/submission-template.md` → edit for your destination
- Commit: `feat: add destination-name`

## 3. Research Requirements

1. **Minimum 3 Independent Sources** — include ≥1 official/institutional
2. **Seasonal Breakdown** — Best / Moderate / Avoid for every season
3. **Bug Category Matrix** — all 7 categories (mosquitoes, sandflies, ticks, spiders, wasps, cockroaches, fleas)
4. **12-Month Freshness** — verify all claims within 12 months
5. **Climate-Factor Reasoning** — explain WHY bugs are absent
6. **Budget & Visa Info** — practical planning essentials

## 4. Open Pull Request

- Clear summary of changes
- Include verification checklist in PR description
- Expect 1–3 review rounds

## 5. Review Criteria

| Criterion | Pass? |
|---|---|
| Source accessibility | All 3 reachable? |
| Bug matrix completeness | 7 categories filled? |
| Seasonal coverage | Best/Moderate/Avoid present? |
| Markdown formatting | Clean tables? No broken links? |
| Community tone | Anti-bug but pro-ecosystem? |

## 6. Ongoing Care

- Update "Last verified" quarterly in volatile regions
- Monitor climate-change insect migration trends
- All calendars: annual re-calibration
